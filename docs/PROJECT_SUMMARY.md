# HinH (Hackathon in Hackathon) — 项目总结文档

## 一、产品定位

HinH 是一款 **AI 驱动的黑客松模拟沙盘**。用户输入 1-4 个创业 idea，40 个拥有独立人格的 AI 角色（OC）自动组队、讨论、创造产品方案、接受评委答辩，最终产出结构化 BP 文档和专业评分。

**核心价值**：将粗略想法通过多角色、多视角的推演碰撞，打磨为有商业价值的成熟方案。

---

## 二、完整玩法流程

### 入口：Idea 输入

- 用户在首页输入 1-4 个 idea
- 1 个 idea → 分发给 4 组各自独立推演（同题竞技）
- 多个 idea → 轮询分配给不同组
- 点击「开始模拟」启动全流程

### Phase 0：自动分组（无对话）

- 40 个 OC 角色按 MBTI 性格匹配岗位（产品经理、前端、后端、设计师、运营）
- 分为多组，每组 2-6 人随机分配，自动选出队长（ENTJ/ENFJ/ESTJ 等外向领导型优先）
- 前端展示角色从随机散落 → 按组归位的动画过渡

### Phase 1：自由讨论（4 组并行）

1. 队长收到 idea，发言提出方向
2. 组员依次各提一个问题
3. 自由讨论环节（回答、反问、补充）
4. 队长总结拍板，敲定选题和产品方向

### Phase 2：创造产品（4 组并行）

1. 队长分配 BP 写作任务
2. 组员确认分工
3. 各自撰写负责板块（可联网搜索市场数据）
4. 队长汇总生成完整 BP 文档

**BP 结构（7 板块）**：项目名称、问题与痛点、解决方案、目标用户、核心功能、商业模式、竞争优势

### Phase 3：评审答辩（逐组进行）

1. 队长代表团队陈述 BP
2. AI 根据项目内容选择 2-3 位评委提问
3. 队长或对口岗位组员回答
4. 6 位评委独立打分 + 点评 + 改进建议

**评分维度（每项 1-10 分）**：
| 维度 | 说明 |
|------|------|
| 创新性 | idea 的新颖程度和差异化 |
| 现场讲述效果 | BP 陈述的逻辑性和说服力 |
| 完成度 | 方案的完整程度和可执行性 |
| 商业潜力 | 市场空间和商业落地可能性 |
| 技术难度 | 技术壁垒和实现挑战 |

### 结果页

- 4 组按总分排名
- 每组可展开查看：完整 BP 文档、各评委打分详情、评语与改进建议

## 三、角色体系

## 四、技术架构

### 前端
- **框架**：Next.js 16 + React 19 + TypeScript
- **样式**：Tailwind CSS 4 + shadcn/ui
- **状态管理**：Zustand
- **实时通信**：SSE (Server-Sent Events)
- **端口**：4711

### 后端
- **框架**：NestJS 11 + TypeORM + PostgreSQL
- **LLM**：OpenAI 兼容 API（当前接入 CloudDreamAI，模型 auto-v2）
- **联网搜索**：DuckDuckGo（并发 3，带缓存）
- **端口**：4811

### 核心 API

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/simulation/start` | POST | 提交 ideas，创建模拟 |
| `/api/simulation/:id/stream` | SSE | 实时事件流 |
| `/api/simulation/:id/status` | GET | 查询模拟状态 |
| `/api/simulation/:id/result` | GET | 获取最终结果 |

### SSE 事件类型

| 事件 | 说明 |
|------|------|
| `phase_change` | 阶段切换 |
| `message` | 角色发言 |
| `agent_typing` | 打字状态指示器 |
| `tool_call` | 联网搜索调用 |
| `complete` | 模拟结束 |
| `error` | 异常 |

### 并发控制
- Phase 0：单线程算法分组
- Phase 1 & 2：4 组并行（Promise.all）
- Phase 3：逐组串行答辩
- LLM 并发上限：6
- 搜索并发上限：3

## 五、当前 UI 风格（v1 — 暗黑赛博朋克）

### 视觉特征

当前版本采用 **暗黑工业风 + CRT 复古终端** 的视觉语言：

- **底色**：纯黑 `#0a0a0a`，面板 `#1a1a1a`
- **文字**：暖白 `#f0ede6`，灰阶层级 `#666` / `#999`
- **字体**：Space Grotesk（标题） + JetBrains Mono（正文/代码）
- **CRT 扫描线**：全局覆盖的半透明条纹叠加层
- **边框**：1px 紫色/白色细线分隔

### 页面布局

**首页**：全屏暗色背景，居中表单，紫色渐变标题，输入框带紫色边框

**模拟页**：
- 左侧 60%：角色花名册网格（Roster Grid）
  - 网格布局，每格 3:4 比例
  - 角色卡片支持翻转、呼吸动画、聚焦高亮
  - Phase 0 随机散布 → Phase 1 按组归位（FLIP 动画）
- 右侧 40%：聊天侧边栏（Chat Sidebar）
  - 顶部 4 个组 Tab 切换
  - 消息流：头像 + 名字 + 角色标签 + 队长徽章 + 内容
  - 打字指示器：脉冲圆点 + 计时器
  - 未读消息红点

**结果页**：可折叠卡片列表，排名徽章（1ST/2ND/3RD/4TH），展开显示 BP 和评分详情

### 动画系统

| 动画 | 说明 |
|------|------|
| `breathe` | 未激活角色的呼吸灯效果 |
| `activateRitual` | 角色激活时的缩放渐入 |
| `shuffleIn` | Phase 0 随机入场 |
| FLIP transition | Phase 0→1 分组归位 |
| `blink` | 打字指示器省略号闪烁 |

## 六、新版 UI 方向（v2 — Versio 风格）

### 设计理念

新版将从暗黑赛博朋克转向 **Versio 风格**，核心特征：

- **明亮、干净、现代**：抛弃暗黑底色，转向浅色/白色基调
- **大量留白**：呼吸感强，信息密度适中
- **圆润几何**：圆角卡片、柔和阴影、温和的边框
- **精致排版**：大字重标题 + 轻量正文，层级分明
- **柔和配色**：低饱和度主色调，点缀色克制使用
- **微妙动效**：缓入缓出的过渡，非突兀的状态变化
- **卡片化布局**：信息以卡片为单位组织，层次清晰
- **品质感细节**：精致的图标、一致的间距系统、优雅的 hover 状态

### 与现版的关键差异

| 维度 | v1（当前） | v2（Versio） |
|------|-----------|-------------|
| 基调 | 暗黑 #0a0a0a | 明亮 / 浅灰白底 |
| 色彩 | 单色 + 紫色强调 | 低饱和度柔和配色 |
| 字体 | 等宽 JetBrains Mono | 现代无衬线（如 Inter / Plus Jakarta Sans） |
| 边框 | 硬边 1px 线条 | 圆角 + 柔和阴影 |
| 质感 | CRT 扫描线、像素感 | 干净、磨砂玻璃、微妙渐变 |
| 动画 | 赛博闪烁、呼吸灯 | 弹性缓动、平滑过渡 |
| 信息密度 | 密集终端风 | 宽松留白、卡片分组 |
| 头像风格 | 像素动漫 | 保持像素动漫（与 Versio 干净风形成趣味反差） |

### 页面设想

**首页**：白色/浅灰底，居中大标题 + 副标题，卡片式输入区域，主色调按钮，底部装饰性角色头像

**模拟页**：
- 顶部状态栏：简洁 pill 形阶段指示器，进度条而非圆点
- 左侧角色网格：白色卡片 + 柔和阴影，圆角头像，状态用颜色标签区分
- 右侧聊天面板：气泡式消息布局，柔和背景色区分不同角色，优雅的 Tab 切换动画

**结果页**：大号排名数字，卡片式布局，评分维度用条形图/雷达图可视化，评委头像 + 点评以引用块形式展示
